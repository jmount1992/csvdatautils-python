CSVData Module

class csvdatautils.csvdata.CSVData(csvfile: str, mappings: dict = {})[source]

A utility class to provide easy accessibility to data stored within CSV, or CSV-like, files.

__init__(csvfile: str, mappings: dict = {}) → None[source]

Initialises a CSVData object. The CSV file should contain headers in the first row. All other rows are considered data. The headers within the CSV file will become the set of accessible fields.

Numeric data will be stored as floats, all other data will be stored as strings except for the strings ‘’ and ‘none’ (or any variation: ‘NONE’, etc.) which will be stored as None.

Parameters

  • csvfile (str) – the path to the ROSData CSV file

  • mappings (bool, optional) – the field/attribute mappings. Allows the retrieval of the same data value via mutliple names. Format is {‘mapped_name’: ‘<field_name>’}. Defaults to {}.

get_subset(indices: Optional[Union[int, List]] = None, fields: Optional[Union[str, List]] = None, operate_on_chunk: bool = False) → Any[source]

Gets a subset of the data. The subset can be a desired set of row(s) and/or field(s).

Examples

# return data for index 0
data = csvrosdata_obj.get_data(0)
# return all pos_x data
data = csvrosdata_obj.get_data(‘pos_x’)
# return all fields for multiple indices
data = csvrosdata_obj.get_data([0, 2])
# return all data for a set of fields
data = csvrosdata_obj.get_data([‘pos_x’, ‘pos_z’])
# return multiple fields for a specified index or a set of indices
data = csvrosdata_obj.get_data(0, [‘pos_x’, ‘pos_z’])
data = csvrosdata_obj.get_data([0, 2], [‘pos_x’, ‘pos_z’])
Parameters

  • indices (int, str, list) – the index(s) to be retrieved

  • fields (optional, int or str) – the field(s) to be retrieved

  • operate_on_chunk (bool, optional) – perform the operation on current chunk. Defaults to False.

Raises

ValueError – if too many arguments are provided

Returns

either the data (list) for a given index, the value for a given index/field or the data (list) for a field across all indices.
